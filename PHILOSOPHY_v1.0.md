# 🟦 FreeLang 철학 선언문 v1.0
## — 소프트웨어 주권 기반 언어 —

## 1️⃣ 핵심 선언

나는 하드웨어 주권을 추구하지 않는다.
나는 **소프트웨어 주권**을 추구한다.

## 2️⃣ 소프트웨어 주권의 정의

소프트웨어 주권이란:

- 코드의 실행 의미가 외부 런타임에 의해 결정되지 않는다.
- 코드의 생명주기가 특정 플랫폼에 종속되지 않는다.
- 실행 통제권이 외부 프레임워크에 넘어가지 않는다.
- 빌드·실행·배포 흐름이 외부 생태계에 묶이지 않는다.
- 핵심 기능은 내부 구현을 우선한다.

## 3️⃣ 철학의 본질

```
내가 코딩하고,
내가 실행하고,
내가 통제한다.
```

외부 도구는 사용할 수 있다.
**그러나 의미론과 실행 주권은 넘기지 않는다.**

## 4️⃣ 의존성에 대한 원칙

- 물리 기반 의존 (CPU, OS)은 인정한다.
- 그러나 소프트웨어 계층의 종속은 최소화한다.
- 외부 라이브러리는 선택이며, 필수 전제는 아니다.
- 대체 불가능한 의존은 구조적으로 배제한다.

## 5️⃣ 구현 방향성

### 🟦 메모리
메모리 정책은 내부에서 정의한다.
외부 런타임에 생명주기를 맡기지 않는다.

### 🟦 타입
타입 시스템은 자체 의미론을 따른다.
타 언어 호환보다 내부 일관성을 우선한다.

### 🟦 동시성
동시성 모델은 외부 런타임 스케줄러에 종속되지 않는다.
실행 흐름은 언어 내부에서 통제한다.

### 🟦 에러 처리
실패의 의미는 언어가 정의한다.
외부 예외 모델을 그대로 복제하지 않는다.

## 6️⃣ 확장이 발생하는 이유

의존을 제거하면 내부 구현은 늘어난다.
**이 확장은 욕심이 아니라 필연적 흡수 과정이다.**

외부에 있던 기능을 내부로 가져오는 것일 뿐이다.

## 7️⃣ 이 철학의 목표

- ✅ 실행 주권 유지
- ✅ 예측 가능한 동작
- ✅ 장기적 안정성
- ✅ 외부 생태계 붕괴와 무관한 지속성

## 8️⃣ 최종 정의

**FreeLang은:**

> "소프트웨어 실행 주권을 보존하기 위한 언어"

이다.

❌ 확장 지향이 아니다.
✅ **의존 회수 지향**이다.
