# 🟦 FreeLang 철학 선언문 v1.0
## — 소프트웨어 주권 기반 언어 —

## 1️⃣ 핵심 선언

나는 하드웨어 주권을 추구하지 않는다.
나는 **소프트웨어 주권**을 추구한다.

## 2️⃣ 소프트웨어 주권의 정의

**외부 의존이 없다는 것:**
> "타 앱/언어/런타임을 사용하지 않고도 우리 언어로 가능한 모든 것을 할 수 있다"

구체적으로:

- 🟦 **완결성**: 컴파일, 실행, 배포까지 FreeLang 자체로 가능
- 🟦 **자급자족**: Python, Rust, Node.js 런타임에 종속 거부
- 🟦 **자체 구현**: 컴파일러, VM, 표준 라이브러리 모두 내부 개발
- 🟦 **통제권 유지**: 실행 의미와 생명주기를 언어가 정의
- 🟦 **독립성**: 외부 도구/프레임워크 없이도 동작

### 허용되는 의존
✅ **OS/CPU/Hardware** - 물리 계층은 기본 전제
✅ **외부 LLM** - AI 도구로서만 (생성 자동화)

### 거부되는 의존
❌ **Python 런타임** - 위에서 실행 불가
❌ **Rust 컴파일러** - 번역 의존 불가
❌ **Node.js** - JavaScript 환경 의존 불가
❌ **타 언어 라이브러리** - 외부 앱 사용 불가

## 3️⃣ 철학의 본질

```
내가 코딩하고,
내가 실행하고,
내가 통제한다.
```

외부 도구는 사용할 수 있다.
**그러나 의미론과 실행 주권은 넘기지 않는다.**

## 4️⃣ 의존성에 대한 원칙

### ✅ 인정하는 의존
- **물리 계층**: CPU, OS, Hardware (불가피)
- **외부 AI**: LLM (Claude API 등) - 생성 도구로만

### ❌ 거부하는 의존
- **타 언어 런타임**: Python, Rust, Node.js 등에 종속 거부
- **외부 도구**: 타 앱 없이도 완전히 동작 가능해야 함
- **번역 의존**: 다른 언어로 컴파일/변환되어 실행 불가
- **대체 불가능한 종속**: 단 하나의 외부 시스템에만 의존 구조 배제

### 📋 언어 완결성 체크리스트
- ✅ 자체 컴파일러 구현
- ✅ 자체 VM 또는 런타임 구현
- ✅ 자체 표준 라이브러리 구현
- ✅ 외부 앱 없이 빌드, 실행, 배포 가능

## 5️⃣ 구현 방향성

### 🟦 메모리
메모리 정책은 내부에서 정의한다.
외부 런타임에 생명주기를 맡기지 않는다.

### 🟦 타입
타입 시스템은 자체 의미론을 따른다.
타 언어 호환보다 내부 일관성을 우선한다.

### 🟦 동시성
동시성 모델은 외부 런타임 스케줄러에 종속되지 않는다.
실행 흐름은 언어 내부에서 통제한다.

### 🟦 에러 처리
실패의 의미는 언어가 정의한다.
외부 예외 모델을 그대로 복제하지 않는다.

## 6️⃣ 확장이 발생하는 이유

의존을 제거하면 내부 구현은 늘어난다.
**이 확장은 욕심이 아니라 필연적 흡수 과정이다.**

외부에 있던 기능을 내부로 가져오는 것일 뿐이다.

## 7️⃣ 이 철학의 목표

- ✅ 실행 주권 유지
- ✅ 예측 가능한 동작
- ✅ 장기적 안정성
- ✅ 외부 생태계 붕괴와 무관한 지속성

## 8️⃣ 최종 정의

**FreeLang은:**

> "타 앱/런타임을 사용하지 않고도,
> 우리 언어 자체로 가능한 모든 것을 할 수 있는 언어"

즉:
- **완전 독립형**: Python, Rust, Node.js 런타임 종속 거부
- **자급자족형**: 컴파일러, VM, 라이브러리 모두 내부 구현
- **통제형**: 실행 의미와 생명주기를 언어가 정의
- **장기 지속형**: 외부 생태계 붕괴와 무관하게 동작

❌ 확장 지향이 아니다.
✅ **완결성 추구**이다.
